# 추론 예시 (MoE) — B5와 비교

`02_model/predict_moe.py`로 **ThreeExpertMoE** 모델을 돌린 결과입니다.
B5(최종 채택)와의 차이는 **게이트 가중치**(각 축을 얼마나 반영했는지)를 볼 수 있다는 점입니다.

> **주의**: MoE는 최종 모델이 **아닙니다**. B5(0.362) > MoE(0.343), p=0.0013으로 B5가 유의하게
> 우위입니다. 이 문서는 MoE 게이트의 동작을 보여주는 **참고용**입니다.
> 실제 추천은 `predict.py`(B5)를 쓰세요.

**게이트 읽는 법**: `gate(cook/flav/prop)` = 세 값의 합이 1. 큰 쪽이 그 조합에서 주력으로
쓰인 축입니다.

---

### 여기서 보이는 것 ★

**게이트가 거의 다 cook(조리)에 쏠려 있습니다** (0.5~0.96). flavor·prop은 대부분 0.2 미만입니다.
이것이 리포트의 "게이트 붕괴" — MoE가 재료마다 다른 축을 고르길 기대했지만, 실제로는 3축이 다
있어도 조리 축을 주력으로 씁니다.

**단 하나의 예외가 용안육(12위, gate=[0,0,1])**입니다. 용안육은 RecipeDB·FlavorGraph에 없고
성미귀경만 있어(`axes=1`) 전통 축이 100%입니다. 대추+용안육은 실제 한방 약선 조합인데,
**MoE가 이 조합에서만 전통 축을 온전히 씁니다.** 전통 축이 살아나는 유일한 자리입니다.

**prop 게이트가 전부 0.00**입니다 — 월계수는 본초 미수재라 전통 축이 없고, 게이트가 이를 정확히
반영해 cook/flavor에만 배분합니다. 셀러리·올리브오일·파슬리처럼 **flavor 게이트가 우세한 조합**도
보입니다. B5(단순 MLP)와 달리 MoE는 "어느 축으로 이 점수가 나왔나"를 명시적으로 드러냅니다.

## 예시 3. JSON — 게이트 포함 (MoE)

```
python predict_moe.py --herb 대추 --candidate 시나몬 --json
```

```json
{
  "herb": "대추", "model": "moe_final",
  "recommendations": [{
    "candidate": "시나몬", "score": 0.949,
    "llm_scores": { "claude": 0.963, "llama": 0.950, "gpt": 0.935 },
    "gate": { "cook": 0.888, "flavor": 0.015, "prop": 0.097 },
    "axes": { "cook": true, "flavor": true, "prop": true }, "n_axes": 3
  }]
}
```

B5 JSON과 차이: `features` 대신 **`gate`**가 있습니다. 대추+시나몬은 cook 0.888로, 이 조합의
근거가 주로 조리 축(RecipeDB 동시등장)임을 보여줍니다.

---

## B5 vs MoE — 같은 입력, 다른 출력

`대추 --candidate 시나몬` 기준:

| | B5 (predict.py) | MoE (predict_moe.py) |
|---|---|---|
| score | 0.955 | 0.949 |
| 근거 제공 | `features`(축별 값) | `gate`(축별 반영 비율) |
| cook 근거 | cook_pmi=0.226 | gate_cook=0.888 |
| 해석 | 어느 feature가 컸나 | 어느 축을 썼나 |

**점수는 거의 같습니다** (0.955 vs 0.949). 순위도 대동소이합니다. 차이는 설명 방식입니다 —
B5는 raw feature 값, MoE는 축별 게이트.

### 그런데 왜 B5를 택했나

1. **성능**: B5 0.362 > MoE 0.343 (p=0.0013, 유의)
2. **게이트가 붕괴**: MoE 게이트가 herb·재료 무관하게 조리로 쏠려(예시 1), "재료별 전문가
   혼합"이라는 해석 이득이 실제로 약함
3. **복잡도**: MoE는 파라미터·구조가 더 복잡한데 이득이 없음

**결론**: MoE의 게이트는 "용안육=전통 100%" 같은 극단 사례에서만 의미 있고, 대부분은 조리로
붕괴합니다. 배포는 B5, MoE는 게이트 분석용으로만 보관합니다.
